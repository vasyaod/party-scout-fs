# MODEL.md ‚Äî JSON data model

The canonical shape of the data the site and clients consume. Files are
generated by `party-scout-code` ‚Äî do not hand-edit; change the generator.

## Layout (partitioned by city)

```
data/
  cities.json                       list of cities (site default = first)
  <city>/index.json                 that city's weeks, newest first
  <city>/<week_start>.json          one week (machine-readable)
  <city>/<week_start>.md            same week, human-readable digest
```

e.g. `data/san-francisco/2026-06-22.json`, `data/los-angeles/2026-06-29.json`.

## `data/cities.json`

```json
{
  "cities": [
    { "slug": "san-francisco", "label": "San Francisco" },
    { "slug": "los-angeles",   "label": "Los Angeles" }
  ]
}
```

The site reads this first to build the city dropdown; order matters (first = default).

## `data/<city>/index.json`

List of that city's weeks, newest first. The site reads it to build the week dropdown.

```json
{
  "weeks": [
    {
      "week_start": "2026-06-22",
      "window": "2026-06-25..2026-06-28",
      "title": "SF Pride weekend",
      "counts": { "music": 19, "sports": 10 }
    }
  ]
}
```

## `data/<city>/<week_start>.json` ‚Äî one week

Named after the **Monday** of the week the window falls in (e.g. the Thu‚ÄìSun
Jun 25‚Äì28 2026 weekend ‚Üí `2026-06-22.json`). Carries `city` + `city_label` too.

```json
{
  "week_start": "2026-06-22",
  "window": "2026-06-25..2026-06-28",
  "title": "SF Pride weekend",
  "location": "San Francisco & Bay Area",
  "city": "san-francisco",
  "city_label": "San Francisco",
  "generated_at": "2026-06-25",
  "tracks": {
    "music":  [ <event>, ... ],
    "sports": [ <event>, ... ]
  }
}
```

| Field | Type | Notes |
|-------|------|-------|
| `week_start` | string `YYYY-MM-DD` | Monday of the week (ISO). Names the file. |
| `window` | string | Inclusive day window, `YYYY-MM-DD..YYYY-MM-DD`. |
| `title` | string | Short **3‚Äì4 word** label for the week (e.g. `"SF Pride weekend"`). Required; no parentheticals. Shown in the site week picker. See REQUIREMENTS rule 3a. |
| `city` / `city_label` | string | City slug (`san-francisco`) + display label (`San Francisco`). Names the `data/<city>/` partition. |
| `location` | string | Default `"San Francisco & Bay Area"`. |
| `generated_at` | string `YYYY-MM-DD` | When the report was built. |
| `tracks.music` / `tracks.sports` | array&lt;event&gt; | The two ranked tracks. |

## `<event>`

```json
{
  "id": "2026-06-22-music-dayfall-karizma-colette-mr-v-7d2a1c",
  "eid": "dayfall-karizma-colette-mr-v-7d2a",
  "track": "music",
  "category": "House",
  "name": "DAYFALL: Karizma, Colette, Mr. V",
  "day": "Sat",
  "date": "2026-06-27",
  "time": "6‚Äì10pm",
  "area": "SF",
  "venue": "The Pearl",
  "address": "601 19th St, San Francisco, CA 94107",
  "why": "Soulful-house heavyweights at sunset on The Pearl rooftop",
  "summary": "DAYFALL brings deep/soulful-house veterans Karizma, Colette and Mr. V to The Pearl's rooftop for a sunset session bridging day and night.",
  "price": "$71",
  "tags": ["sunset", "outdoor", "house", "deep-house", "disco-house", "soulful"],
  "sources": ["https://ra.co/events/2463690"],
  "tickets": ["https://www.eventbrite.com/e/dayfall-between-day-night-tickets-1989860033927"],
  "maps": {
    "google": "https://www.google.com/maps/search/?api=1&query=The+Pearl%2C+601+19th+St...",
    "apple":  "https://maps.apple.com/?q=The+Pearl%2C+601+19th+St..."
  },
  "ra": { "event_id": "2463690", "url": "https://ra.co/events/2463690", "interested": 1, "attending": 1 },
  "image": "https://api.party-scout.app/img/san-francisco/2026-06-22/dayfall-karizma-colette-mr-v-7d2a.jpg",
  "image_generated": false,
  "popularity": 7,
  "active": true
}
```

| Field | Type | Notes |
|-------|------|-------|
| `id` | string | **Stable** per event/week ‚Äî derived from week + track + name/day/area/venue. Used for **MERGE** matching; survives re-runs so the same event is tracked over time. |
| `eid` | string | Short, URL-safe, week-unique **selection id** (e.g. `kaskade-90bf`). For UI deep-linking only ‚Äî the card's DOM id and the `/#<eid>` anchor. **Not** used for merging. |
| `track` | string | `"music"` or `"sports"`. |
| `category` | string | Sub-genre/type label, e.g. `House`, `Moto`. |
| `name` | string | Event name (shown as the card title; not a link). |
| `day` | string | `Fri` / `Sat` / `Sun` / `Sat/Sun` etc. |
| `date` | string `YYYY-MM-DD` | The event's **calendar date**, derived from `week_start` (the Monday) + `day`. For a multi-day `day` (`Sat/Sun`) it's the **first** day. `""` when `day` is missing/unparseable. Generated ‚Äî not hand-set. |
| `time` | string | Free-form, e.g. `6‚Äì10pm`, `9pm`, `all-day‚Äì3am`. |
| `area` | string | City/neighborhood, e.g. `SF`, `Oakland`, `Sacramento`. |
| `venue` | string | Venue name; `""` if unknown. |
| `address` | string | Exact street address (city/state) when known ‚Üí precise Maps pin. |
| `why` | string | **Differentiator** one-liner (~8‚Äì16 words) ‚Äî the reason to pick THIS event over others that weekend; never restates the lineup/title/process (that's `summary`). Shown on the card. See REQUIREMENTS rule 18. |
| `summary` | string | 1‚Äì2 sentence event summary, AI-written from the event's actual page (read via an ephemeral remote-browser session). |
| `price` | string | Lowest/GA price `"$NN"`, a range `"$NN‚ÄìMM"`, or `"Free"`. `""` if unobtainable (never guessed). |
| `tags` | array&lt;string&gt; | Genre/vibe tags (house, techno, free, outdoor, rooftop, pride, daytime‚Ä¶). Powers the tag cloud + filter. |
| `sources` | array&lt;string&gt; | **Ordered list of the event's info page URLs** (replaces the old single `link`) ‚Üí the card's **Open** button uses **index 0** (the highest-priority link). Scrape sources (19hz) are dropped; auto-filled from the RA/ticketer page when no explicit link. `[]` when none known. Merges as an order-preserving union (first wins). |
| `tickets` | array&lt;string&gt; | **Ordered list of ticket-seller URLs** (RA/Tixr/Eventbrite/Etix/Ticketmaster/AXS/‚Ä¶) ‚Üí **Buy ticket** uses **index 0**. Never a search engine. The same URL may also appear in `sources` (an event page can be both info + ticket). Merges as an order-preserving union. |
| `maps` | object | `{ "google", "apple" }` Maps URLs (from `address` ‚Ä∫ known-venue table ‚Ä∫ `venue, area`). Client opens the right app per device. |
| `ra` | object | Resident Advisor rating: `{ event_id, url, interested, attending }`. `{}` when the event isn't on RA. Popularity signal. |
| `image` | string | Public URL of the event's image ‚Äî the flyer/hero (`og:image`, ticket-site preferred), an AI-generated placeholder, else `""` (the UI then shows the stub). A **‚â§512px JPG**. Filled by `enrich_images.py`. See REQUIREMENTS rule 20a. |
| `image_generated` | boolean | `true` when `image` is an **AI-generated placeholder** (no real flyer could be found) ‚Äî so it can be **replaced** if a real flyer turns up later. `false` for a real found flyer (or no image). Set by the enrichment, preserved across re-generation. |
| `popularity` | integer 0‚Äì10 | **Hidden** heuristic popularity/hype score (NOT shown on the site) ‚Äî a ranking signal. Blended from RA interest count, headliner/marquee fame, festival/block-party scale, and price tier. |
| `active` | boolean | `true` if present in the latest scan; `false` = "carried over" (kept from a prior scan, dimmed on the site). Never deleted. |

### Link rules (summary ‚Äî see REQUIREMENTS.md)

- `tickets`, `maps`, `sources` are **never shown as raw URL text** ‚Äî only as buttons /
  the `open map` link. `tickets[0]` ‚Üí **Buy ticket** (real seller only). `sources[0]` ‚Üí
  **Open** (event page). `maps` ‚Üí the üìç **open map** link (Apple on iOS/iPadOS/
  macOS, Google elsewhere).
- `price` is combined into the Buy-ticket button (`$NN ¬∑ Buy ticket`); `Free`
  renders as a gold non-clickable button.
