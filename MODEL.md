# MODEL.md — JSON data model

The canonical shape of the data the site and clients consume. Files are
generated by `party-scout-code` — do not hand-edit; change the generator.

## `data/index.json`

List of all weeks, newest first. The site reads this to build the week dropdown.

```json
{
  "weeks": [
    {
      "week_start": "2026-06-22",
      "window": "2026-06-25..2026-06-28",
      "title": "SF Pride weekend (stacked)",
      "counts": { "music": 19, "sports": 10 }
    }
  ]
}
```

## `data/<week_start>.json` — one week

Named after the **Monday** of the week the window falls in (e.g. the Thu–Sun
Jun 25–28 2026 weekend → `2026-06-22.json`).

```json
{
  "week_start": "2026-06-22",
  "window": "2026-06-25..2026-06-28",
  "title": "SF Pride weekend (stacked)",
  "location": "San Francisco & Bay Area",
  "generated_at": "2026-06-25",
  "tracks": {
    "music":  [ <event>, ... ],
    "sports": [ <event>, ... ]
  }
}
```

| Field | Type | Notes |
|-------|------|-------|
| `week_start` | string `YYYY-MM-DD` | Monday of the week (ISO). Names the file. |
| `window` | string | Inclusive day window, `YYYY-MM-DD..YYYY-MM-DD`. |
| `title` | string | Human label for the week. |
| `location` | string | Default `"San Francisco & Bay Area"`. |
| `generated_at` | string `YYYY-MM-DD` | When the report was built. |
| `tracks.music` / `tracks.sports` | array&lt;event&gt; | The two ranked tracks. |

## `<event>`

```json
{
  "id": "2026-06-22-music-dayfall-karizma-colette-mr-v-7d2a1c",
  "track": "music",
  "category": "House",
  "name": "DAYFALL: Karizma, Colette, Mr. V",
  "day": "Sat",
  "time": "6–10pm",
  "area": "SF",
  "venue": "The Pearl",
  "address": "601 19th St, San Francisco, CA 94107",
  "price": "$71",
  "tags": ["sunset", "outdoor", "house", "deep-house", "disco-house", "soulful"],
  "link": "https://ra.co/events/2463690",
  "tickets": "https://www.eventbrite.com/e/dayfall-between-day-night-tickets-1989860033927",
  "maps": {
    "google": "https://www.google.com/maps/search/?api=1&query=The+Pearl%2C+601+19th+St...",
    "apple":  "https://maps.apple.com/?q=The+Pearl%2C+601+19th+St..."
  },
  "ra": { "event_id": "2463690", "url": "https://ra.co/events/2463690", "interested": 1, "attending": 1 },
  "popularity": 7,
  "active": true
}
```

| Field | Type | Notes |
|-------|------|-------|
| `id` | string | **Stable** per event/week — derived from week + track + name/day/area/venue. Survives re-runs so the same event can be tracked over time. |
| `track` | string | `"music"` or `"sports"`. |
| `category` | string | Sub-genre/type label, e.g. `House`, `Moto`. |
| `name` | string | Event name (shown as the card title; not a link). |
| `day` | string | `Fri` / `Sat` / `Sun` / `Sat/Sun` etc. |
| `time` | string | Free-form, e.g. `6–10pm`, `9pm`, `all-day–3am`. |
| `area` | string | City/neighborhood, e.g. `SF`, `Oakland`, `Sacramento`. |
| `venue` | string | Venue name; `""` if unknown. |
| `address` | string | Exact street address (city/state) when known → precise Maps pin. |
| `price` | string | Lowest/GA price `"$NN"`, a range `"$NN–MM"`, or `"Free"`. `""` if unobtainable (never guessed). |
| `tags` | array&lt;string&gt; | Genre/vibe tags (house, techno, free, outdoor, rooftop, pride, daytime…). Powers the tag cloud + filter. |
| `link` | string | **Event website / info page** → the card's **Open** button. Auto-filled from RA/ticketer page when not explicit. |
| `tickets` | string | Direct **ticket seller** URL (RA/Tixr/Eventbrite/Etix/Ticketmaster/AXS/…) → **Buy ticket** button. Never a search engine. |
| `maps` | object | `{ "google", "apple" }` Maps URLs (from `address` › known-venue table › `venue, area`). Client opens the right app per device. |
| `ra` | object | Resident Advisor rating: `{ event_id, url, interested, attending }`. `{}` when the event isn't on RA. Popularity signal. |
| `popularity` | integer 0–10 | **Hidden** heuristic popularity/hype score (NOT shown on the site) — a ranking signal. Blended from RA interest count, headliner/marquee fame, festival/block-party scale, and price tier. |
| `active` | boolean | `true` if present in the latest scan; `false` = "carried over" (kept from a prior scan, dimmed on the site). Never deleted. |

### Link rules (summary — see REQUIREMENTS.md)

- `tickets`, `maps`, `link` are **never shown as raw URL text** — only as buttons /
  the `open map` link. `tickets` → **Buy ticket** (real seller only). `link` →
  **Open** (event page). `maps` → the 📍 **open map** link (Apple on iOS/iPadOS/
  macOS, Google elsewhere).
- `price` is combined into the Buy-ticket button (`$NN · Buy ticket`); `Free`
  renders as a gold non-clickable button.
